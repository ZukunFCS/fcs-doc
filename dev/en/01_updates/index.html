FCS 26.04.01

  • Date: 2026/04/01

  • Version: 26.04.01

  • Stage: Stable

26.04 is the largest release to date. We have added many new features and improvements to help you get more out of FCS.

Text unverified

Additions / New Features

  • CLI support (script runner + schema) and addition of sample/test scripts (Docs)

    A CLI tool has been added for users who want to integrate FCS into their in-house workflows and tools, allowing FCS to be operated from scripts.
    You can process videos, upload animations, and more from scripts without launching the UI.
    The CLI tool is bundled with FCS and is available to all FCS users at no additional cost.
    Comprehensive documentation for the CLI tool is provided, so please feel free to contact us if you need help integrating it into your workflow.

  • Gallery loading optimization (thumbnail support)

    Profile images are now saved as thumbnails alongside the original images, significantly speeding up startup especially for large sessions.
    Startup is approximately 2x faster compared to the previous version.

  • Interactive UI documentation (experimental feature)

    An experimental UI document powered by NotebookLM has been added.
    You can ask questions about FCS windows and workflows.
    Access it here.

  • New Pipelines

    Significantly improved the stability and accuracy of tracking algorithms.
    The new pipelines (Rich++, RP++, Robust++) retain their respective characteristics while offering improved overall performance and accuracy.
    Internal benchmarking of the new pipelines against previous versions confirmed significant performance improvements.
    For details on the new pipelines and evaluation methods, see here.

  • Visual Analysis Tool

    A new document has been added that allows you to quickly review tracking results across various scenarios.
    It helps you quickly determine whether FCS is suitable for your project and how much time is needed to achieve your desired quality.

Changes / Improvements

  • UI overhaul and stability improvements

    The UI has been overhauled with significant improvements to stability and performance.
    The menu bar now also functions as a status bar, displaying current session information.

  • LM frame export is not supported in the ++ workflow and has been deprecated.
    It is scheduled for removal in 26.10. Please let us know if this affects your workflow.

  • When FCS crashes or encounters an error, you can now report it simply by clicking a button in the UI.
    This information is used to improve FCS and quickly fix issues.

  • Fullscreen mode has been removed as it was not used by many users.

  • The in-memory file system has been removed as it did not achieve the expected performance gains.

  • Long-running Maya commands (such as opening scenes) no longer block the UI and are now executed separately in the background.

  • Log messages are now displayed at the bottom of the UI instead of the top.
    Clicking opens the log window (left-click), and copies the message to the clipboard (right-click).

Fixes

  • Video player: Fixed frame index exceeding total frame count

    A failsafe has been added to prevent cases where the frame index exceeded the total frame count with certain encodings, which could cause the program to crash.

  • Video player: Fixed audio popping noise

    Fixed an issue where popping noise occurred when playing audio in FCS.

  • Fixed timeline / UI resizing issues and plot / cursor alignment bugs

    Fixed issues where the timeline window did not resize correctly and the cursor did not align properly with the timeline.
